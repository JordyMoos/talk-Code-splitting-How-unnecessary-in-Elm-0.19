So elm 0.19 what is in in for us? Lets go over the highlights.
First the main selling point: Small asset size. Elm 0.19 compiler does a few tricks to get the compiled code to be really small. The “real world app” of 2.000 lines of code is only 29kb. Which is way smaller than some of the competition.

The cool thing is that with an example project with 50.000 lines compiled to 114kb of size which is just a bit bigger than the competition with just 2.000 lines of code.

All we have to do is compile with the --optimize flag and then the minify/gzip part.
An other selling point is that we got faster compile times.
List and Dicts got faster. Which is really nice if you would be making a tile game.
These were for me the biggest selling points, now let’s take a look of the things that changed and how to deal with them.
The command line is a bit renamed but generally does the same thing.

elm-package.json -> elm.json

Elm-package.json has been renamed to elm.json
And the structure is a bit different depending if you have an application or a package.
The most important change for applications is that you now specify an explicit version for you dependencies. Instead of a range. A benefit for that approach is that the elm.json also acts as a lock file. And the dependencies are split into “direct” and “indirect”. You just put all your dependencies in “direct”. And “indirect” will list your dependencies dependencies.

And you can also list your test dependencies here. Which removes the need of a separate elm.json in your test directory.
Some functions have changed, all with the idea to make the language easier and less error prone. For that reason Basics.toString has been moved to Debug.toString to indicate that you should not be using it. Most of the time you would be transforming an int or float to a string to display it or something and for that you must use the new String.fromInt and String.fromFloat which are more explicit.
You can no longer create custom infix operators. Because they tend to make your code less readable. And some core infix operators are also gone. Like `%` is replaced with `modBy` and `Platform.Cmd.(!)` has been removed entirely. You now must return the model / cmd tuple explicitly.
  • % -> modBy
  • Platform.Cmd.(!) -> (model, cmd)
  • uncurry, curry, flip -> gone
There is a new package elm/browser which is a combination of some older packages. Like mouse and the dom. And also the new starting point of your application instead of Html.program.
Since 0.19 you can no longer use native modules. Easy said that is javascript code pretending to be elm. It is something we got warned for about a year ago and caused lots of discussions on the internet. Because people are relying on it. Though by removing native support, elm is able to do better optimization with dead code removal. And can keep its promise to not cause runtime errors. Only the elm-lang and elm-explorations organizations can still use native code.
That was a speedy tour to the mayor changes
All in all the api changes are not that bad let’s do a small upgrade together.
